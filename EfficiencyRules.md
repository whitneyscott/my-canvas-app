# 🚀 CURSOR 2026: MAXIMUM OUTPUT, MINIMUM COST

### 1. THE MODE LADDER (Select before you type)
* **ASK:** Questions/Syntax only. (Cheapest/No file edits).
* **PLAN:** Architecture & Logic. Always use this first for features. Review the text plan to catch errors before paying for code.
* **AGENT:** Execution. Use ONLY to apply a finished plan or run terminal commands. (Most expensive).
* **DEBUG:** Error fixing. Use only when you have a specific crash/bug.

---

### 2. THE "GRUG" PROMPT RULES (Save Tokens)
* **Fresh Start (Ctrl + L):** Clear chat for every new task. Don't carry "Auth" history into "Database" work.
* **No @Codebase:** Use `@Files` or `@Folders`. Scanning the whole project is a "token nuke."
* **Surgical Edits:** If the AI rewrites a whole file, stop it. Say: "Diff only."
* **One Step:** Ask for one file change at a time. Multiple changes in one prompt lead to expensive retries.

---

### 3. MANDATORY 2026 SETTINGS
* **Model:** Set to **"Auto"**. Uses cheap models for Plan, Pro models for Agent.
* **Auto-Run:** Set to **"Ask Every Time"**. Prevents infinite loops that drain credits.
* **Context Caching:** Ensure **ENABLED**. Saves ~90% on repeated file reads.
* **Hard Limit:** Set a $10-$20 monthly cap in your Cursor Web Dashboard.
